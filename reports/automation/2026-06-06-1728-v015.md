## 本次优化建议

1. 新增专用 1200x630 WebP 分享预览图。
   - 原因：分享卡片需要第一眼表达“灰雨电台”的雨夜、屋顶、lofi 和后朋克气质。
   - 影响：提升外部分享识别度，不改变页面布局。
   - 风险：新增一个静态资源，需要确认路径、尺寸和文件体积。
2. 为 `og:image` 补充 type / width / height。
   - 原因：让爬虫更明确地识别图片格式和尺寸。
   - 影响：元信息更完整，构建风险低。
   - 风险：如果默认图片未来替换，需要同步尺寸配置。
3. 继续加固本地截图脚本的清理逻辑。
   - 原因：脚本成功后仍可能残留 preview/Edge profile。
   - 影响：减少下次自动化端口占用和手动清理。
   - 风险：`tools/` 当前为本地忽略目录，不宜直接重新公开提交。
4. 为移动端 contact / reading-card 增加截图点。
   - 原因：目前固定截图覆盖 mobile home 和 blog notes，对联络区移动端覆盖不足。
   - 影响：更容易发现后续小屏布局问题。
   - 风险：只影响自动化检查流程，不影响网站。
5. 后续可补充更轻量的站点身份微文案。
   - 原因：站点已经有视觉系统，下一步可让内容表达更像个人入口。
   - 影响：提高记忆点。
   - 风险：文案过多会稀释当前冷静氛围，需要克制。
