## 本次优化建议

- 移动端导航完整保留：原因是窄屏隐藏入口会降低页面可达性；影响是手机端也能直接进入“档案 / 联络”；风险低，仅改 CSS 布局。
- Hero 深链首屏定位微调：原因是直接打开 hash 页面仍可能受图片和滚动时序影响；影响是分享分区链接更稳定；风险中，需要谨慎处理滚动时机。
- 本地截图脚本清理加固：原因是多轮残留 preview / Edge profile；影响是自动化更稳定；风险低，但 `tools/` 当前为本地忽略目录，不适合默认公开提交。
- 增加 mobile contact / reading-card 截图点：原因是当前脚本移动端覆盖仍偏少；影响是更早发现联络和资料卡移动端问题；风险低。
- CSS 视觉层减重审计：原因是全局 CSS 已超过 3000 行，视觉层较多；影响是降低未来性能和维护风险；风险中，容易影响现有氛围。
