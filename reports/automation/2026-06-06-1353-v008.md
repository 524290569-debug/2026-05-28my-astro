## 本次检查到的问题

- 首页和 Hero 中的 WebP 图片已有稳定资源尺寸，但 `<img>` 未声明 `width` / `height`，存在轻微 layout shift 风险。
- 作品图片和档案桌面图依赖容器高度，缺少明确 `aspect-ratio`，后续内容扩展时更容易产生高度漂移。
- 手机端频道状态栏略高，中央装饰分割线在单列状态下意义不大，容易显得拥挤。
- 执行过程中遇到若干自动化环境兼容问题：`$env:CODEX_HOME` 为空、当前 Git 不支持 `git branch --show-current`、PowerShell `Start-Process` 不允许 stdout/stderr 指向同一文件、Browser 运行时不支持 `networkidle`、截图批量执行曾超时。均已通过既有回退路径、`git rev-parse --abbrev-ref HEAD`、拆分日志、改用 `load` 状态和分批/缓存破除截图处理。
- 初次小屏复检发现 `aspect-ratio` 与旧 `min-height` 叠加后会让作品图卡片被撑宽；已在本轮修复。

## 本次优化建议

- 建议 1：为所有站内 WebP 图片补齐 intrinsic size。原因是减少图片加载前后的布局跳动；影响是稳定 Hero、作品区和档案区排版；风险低，仅增加 HTML 属性。
- 建议 2：为作品卡片和档案图补充 `aspect-ratio`。原因是当前图片资源比例固定，显式比例能让响应式布局更可控；影响是降低图片区域高度漂移；风险低，但需复查手机端最小高度。
- 建议 3：收敛手机端频道状态栏。原因是单列状态下信息块密度偏高；影响是首屏下方过渡更干净；风险低，只改小屏断点。
- 建议 4：后续可优化锚点落位。原因是部分 section 顶部留白较多，截图定位时容易看到上一段尾部；影响是点击导航后的阅读节奏更准确；风险中低，需要整体检查各分区滚动位置。

## 本次实际修改

- 采纳建议 1、2、3，作为同一组“图片稳定尺寸与手机信息密度”低风险优化。
- 为 Hero 三层图、作品三张图、档案桌面图补充 `width` / `height`。
- 为作品图片卡和档案图片卡补充 `aspect-ratio`。
- 在 `max-width: 520px` 下收紧频道状态栏高度、字号和间距，并隐藏单列下无必要的装饰分割线。
- 修正小屏作品图片卡因 `min-height` 造成的宽度外溢：手机窄屏下图片卡和档案图取消最小高度，由比例控制高度。
